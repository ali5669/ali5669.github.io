thttpd源码分析

发表于2024-12-24|更新于2025-01-17|服务端
|阅读量:

前言

thttpd is a simple, small, portable, fast, and secure HTTP server.

组件

组件名 功能
fdwatch 文件描述符管理器👁👁
match 文件名匹配器
mmc 文件内存映射管理
tdate_parse 时间变time_t
timers client定时器
libhttpd 给thttpd提供各种方法
thttpd HttpServer主模块

fdwatch

函数列表

函数名 功能
fdwatch_get_nfiles 获取系统能够拥有多少文件描述符，以及初始化fdwatch数据结构，返回值为-1表示失败
fdwatch_add_fd 在观察列表中添加一个文件描述符
fdwatch_del_fd 在观察列表中删除一个文件描述符
fdwatch 开始观察。返回值是准备好的描述符的数量，如果超时过期则返回0，如果错误则返回-1。超时为INFTIM表示无限期等待。
fdwatch_check_fd 检测某个文件描述符是否可用
fdwatch_get_next_client_data 获取下一个返回事件的客户端数据。当没有事件时返回-1
fdwatch_logstats 生成调试统计syslog信息

match

函数列表

函数名 功能
match 简单的shell风格的文件名模式匹配器

mmc

函数列表

函数名 功能
mmc_map 返回给定文件的mmap()区域
mmc_unmap 处理mmc_map的返回值
mmc_cleanup 清理mmc包，释放所有未使用的存储空间
mmc_term 清理所有缓存，经常在准备退出时使用
mmc_logstats 生成调试统计syslog信息

tdate_parse

函数列表

函数名 功能
tdate_parse 将输入的时间字符串转换成 time_t 格式

timers

函数列表

函数名 功能
TimerProc 计时器到期时会调用TimerProc。它会收到与计时器相关联的ClientData，以及一个时间值，以防它想要调度另一个计时器。 — 这是一个函数指针
tmr_init 初始化计时器包
tmr_create 初始化一个计时器，是周期性或一次性的
tmr_timeout 返回一个超时，指示距离下一个计时器触发还有多长时间
tmr_mstimeout 毫秒级的timeout
tmr_run 运行计时器列表
tmr_reset 重置一个计时器的时钟，到当前时间加上原本的timeout
tmr_cancel 去除一个计时器
tmr_cleanup 清理计时器包，释放所有没有使用的计时器
tmr_term 清理所有计时器并清除缓存
tmr_logstats 生成调试统计syslog信息

libhttpd

函数列表

对外暴露：

函数名 功能
httpd_initialize 初始化，返回一个包含socket fd的httpd_server
httpd_set_logfp 更改日志文件
httpd_unlisten 关闭socket
httpd_terminate 关闭服务器
httpd_get_conn 当一个监听文件描述符准备好读取时，调用此函数。它会调用accept()并返回一个httpd conn*，其中包含从该文件描述符读取请求和向该文件描述符写入响应的fd。返回accept()是否失败、成功或没有更多连接可接受的指示。为了最小化 malloc()，调用者传入 httpd conn。调用者还负责在第1次调用之前使用每个不同的 httpd conn 将 initialized 设置为零。
httpd_got_request 检查 hc->read buf中的数据是否构成完整的请求。调用者将数据读入hc->read buf【hc->read idx】，并递增 hc->read idx。此例程检查已读取的数据，使用 hc->checked idx和 hc->checked state 进行跟踪，并返回指示是否尚未有完整请求、是否有完整请求或由于语法错误而不会有有效请求的指示。
httpd_parse_request 在hc->read buf中解析请求。在hc中填充大量字段，如URL和各种标题。
httpd_start_request 开始将数据发送回客户端。在某些情况下(目录、CGI程序)，它会自行完成发送。
httpd_write_response 发送任何缓冲的返回文本
httpd_close_conn 调用这个函数来关闭连接并释放数据。这是一个关键点，如果你打开了连接，你仍然应该在父进程中调用这个函数-连接将在子进程中保持打开状态。如果你没有当前的 timeval，只需传入 0。
httpd_destroy_conn 调用此函数来解初始化连接结构并释放已分配的字符串。
httpd_send_err 向客户端返回一个报错
httpd_method_str 生成方法的字符串表示
httpd_realloc_str 初回收字符串
httpd_ntoa 将socket格式化为字符串表示
httpd_set_ndelay 将socket设置为NDELAY模式
httpd_clear_ndelay 清除socket的NDELAY模式
httpd_read_fully 完整读取缓冲区
httpd_write_fully 完整写入缓冲区
httpd_logstats 生成调试统计syslog信息
